## 2025-02-21 - Device Test Infrastructure & Decimation Optimization
**Learning:** The `device/tests` directory was unbuildable due to missing Pico SDK mocks (`pico/multicore.h`, `hardware/timer.h`, `hardware/sync.h`). Also, `device/command_handler.hpp` had a logic bug handling timestamps equal to 0, which caused tests to fail when using 0 as a start time. Additionally, `kMaxTxPayloadLen` was incorrectly set to 1025 instead of 4097, causing buffer overflow assertions in tests.
**Action:** When working with embedded firmware tests on host, always verify they compile and run first. If missing mocks are encountered, create them. When mocking hardware drivers (like `INA228`), ensure the mock file name matches the real file name if the code uses quoted includes, and manage include paths carefully in CMake to avoid shadowing conflicts (e.g. use `mocks/ina228_mock/` separate from general `mocks/`).
